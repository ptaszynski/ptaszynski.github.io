Sign in with a GitHub Personal Access Token

This page is 100% static — there is no backend. To save a new publication, it commits to the GitHub repo via your token, which lives only in your browser (localStorage) and is sent only to api.github.com.

How to create a token (one-time, ~60 seconds)
  1. Go to github.com/settings/personal-access-tokens/new.
  2. Token name: anything (e.g. "site editor").
  3. Resource owner: your account.
  4. Repository access: Only select repositories → pick your homepage repo (e.g. ptaszynski.github.io).
  5. Permissions → Repository → Contents: set to Read and write. Leave the rest default.
  6. Expiration: 90 days is reasonable.
  7. Click Generate token, copy it, paste it below. Treat it like a password.

Click Sign out anytime to wipe the token from this browser. Only the repo owner can hold a token that works on this repo — GitHub enforces that.
